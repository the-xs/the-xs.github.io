<a id="user-content-hot-reload-made-easy-with-webpack" class="anchor" href="#hot-reload-made-easy-with-webpack" aria-hidden="true"><span aria-hidden="true" class="octicon octicon-link"></span></a>Hot reload made easy with webpack</h1>

<p><em>by the-xs</em></p>

<p>At some point, hot reload can help speed up the development when UI and states get complicated. In case needed, there is <a href="http://stackoverflow.com/a/24587740/6013584">post</a> help understand it. So this could save you a lot of time when you want to debug a component with a bunch of state already loaded in the UI; HMR makes the state unchanged but just hot swap the changes you have made to your component.</p>

<p>There are different ways to provide HMR to your project; Some methods might not work for all situation. For example, the stateless functional compoenet might not be able to hot replaced with babel-preset-react-hmre:</p>

<div class="highlight highlight-source-js"><pre><span class="pl-k">const</span> <span class="pl-c1">Widget</span> <span class="pl-k">=</span> () <span class="pl-k">=&gt;</span> <span class="pl-k">&lt;</span>div<span class="pl-k">&gt;</span>hello, word<span class="pl-k">&lt;</span><span class="pl-k">/</span>div<span class="pl-k">&gt;</span></pre></div>

<p>The sympton is console logs HMR is enabled, and components are updated, but browser content isn't updated. Troubleshooting HMR is quite tedious.</p>

<h2>
<a id="user-content-webpack-2-integration-with-react-hot-loader" class="anchor" href="#webpack-2-integration-with-react-hot-loader" aria-hidden="true"><span aria-hidden="true" class="octicon octicon-link"></span></a>Webpack 2 integration with react-hot-loader</h2>
